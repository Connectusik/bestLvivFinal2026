# Довгострокове бачення і roadmap

## Бачення на 3 роки

Перетворити ОТГ-Audit з **інструменту для однієї ОТГ** на
**національну платформу крос-реєстр-аудиту майнових активів**, яка
виявляє корупційні ризики та дірки в місцевих бюджетах до того, як їх
знайде аудит або журналіст.

## Поточний стан (v2.0)

- Завантаження двох Excel/CSV → чистий xlsx з 5 аркушами
- 12 правил нормалізації (адреси, ПІБ, дати, ID)
- 11 детекторів розбіжностей
- Аудит-слід (SHA-256, версії, timestamp)
- Працює локально, ~5 с на 42k рядків
- Один бухгалтер громади вже може приймати пропозицію

## M+3 (стабілізація на реальних даних)

Фокус — точність детекторів на різноманітних датасетах.

- Підключення **реєстру договорів оренди землі** (ще один Excel від
  землевпорядника). Новий детектор: «орендар у договорі ≠ власник у ДЗК».
- Детектор «дата реєстрації нерухомості пізніша за дату припинення» —
  класичний слід data entry error.
- **Fuzzy matching адрес**: "вул. Шевченка 12" ↔ "Шевченка буд. 12"
  (Levenshtein + tokenization).
- Валідація на ще 2 громадах (Білзька, Сокальська).

## M+6 (мобільний додаток для інспекторів)

Мета: зв'язати юридичну розбіжність з фізичною реальністю.

- iOS/Android клієнт для польового інспектора ОТГ.
- Потік: розбіжність в xlsx → інспектор бачить адресу в додатку →
  переходить на місце → фотографує з GPS-міткою → прикріпляє фото до
  розбіжності в центральній базі.
- Фото з GPS + timestamp — **юридичний доказ** для податкового
  повідомлення-рішення.
- Інтеграція з **ortofoto.gov.ua** (ортофотоплани) — порівняння
  «було 2016 → є 2024» для виявлення нових нерухомостей.

## M+9 (інтеграція з ДРАЦС і СПАДЩИНА)

Мета — автоматизувати найбільшу категорію прихованого податку.

- Офіційний доступ до ДРАЦС через API Мін'юсту (ліцензія).
- Детектор: **«власник помер >6 місяців тому, але спадщина не
  переоформлена»** — типова ситуація, коли ОТГ втрачає податок на
  роки через бездіяльність спадкоємців.
- Автоматичне формування листів-попереджень спадкоємцям у рамках
  процедур ст. 73 Земельного кодексу.

## M+12 (ML-скоринг)

Мета — скоротити час бухгалтера на 3× за рахунок пріоритизації.

- ML-модель на основі історичних даних з пілотних громад: «яка
  ймовірність того, що ця розбіжність закінчиться реальним
  донарахуванням».
- Features: тип розбіжності, розмір ділянки, тип нерухомості,
  наявність в інших реєстрах, активність в юрособах.
- Топ-10 % найімовірніших — йдуть першими в роботу інспектора.

## M+18 (крос-громадське погодження)

Мета — одне рішення для кластера з 5–10 сусідніх ОТГ.

- Об'єднаний кабінет для області: ОДА бачить усі громади одночасно.
- Детектор міжкомунальних конфліктів: «людина задекларувала постійне
  проживання в ОТГ А, але володіє нерухомістю тільки в ОТГ Б».
- Рольова модель: голова РДА бачить все, голова ОТГ — тільки свою громаду.

## M+24 (інтеграція з ДПС)

- Пілот з обласною ДПС: двостороння синхронізація даних.
- Сертифікація в Мінцифри як «інтегратор реєстрів».
- Відкриваємо API для журналістів, антикорупційних платформ, Рахункової
  палати.

## M+36 (інша географія, інша тематика)

### Новий ринок
- **Молдова** — найближчий правовий контекст. ERDR + primăria.
- **Грузія** — після Молдови.

### Новий актив
- Реєстр доріг громади ↔ ДТП з Нацполіції ↔ бюджет ремонтів.
- Виявлення доріг, які «ремонтуються» в документах але не в реальності.

### Новий B2G-клієнт
- **Держаудитслужба** — ліцензія на пакетну обробку всіх ОТГ України.
- **Рахункова палата** — верифікація цільового використання бюджетних коштів.

## Потенційні партнерства

1. **Асоціація ОТГ (АОТГ/АМУ)** — доступ до всіх громад.
2. **Prozorro** — наші розбіжності → публікація коригуючих лотів оренди.
3. **YouControl / Opendatabot** — перехресна перевірка з реєстром юросіб.
4. **Мінцифри (Дія.Engine)** — канал до всіх 1 469 ОТГ.
5. **Transparency International Ukraine** — громадська легітимність.
6. **USAID HOVERLA** — грантова підтримка пілотів у західних областях.

## Чого ми **не** робимо (і чому)

- **Національний реєстр реєстрів** — робота Мінцифри; ми додаємо вартість
  поверх державних реєстрів, а не замінюємо їх.
- **Автоматичні податкові повідомлення-рішення** — це повноваження
  виключно ДПС. Ми готуємо дані, рішення приймає людина.
- **Правоохоронний блок (НАБУ, ДБР)** — інший етичний режим, інший ринок,
  не мішаємо.
